// APPROACH

I try to be useful from day one and forgettable on day-last — leaving the team with patterns and habits, not a designer-shaped dependency.

  • Audit, listen, then propose. The first week is mostly questions.
  • Ship something small early to test the working rhythm with the team.
  • Document decisions as I go so the team can keep moving after I roll off.
  • Set a clear end-state up front — what "done" looks like — so the engagement doesn't drift.
