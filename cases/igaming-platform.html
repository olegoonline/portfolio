iGaming Platform — Multibrand Design System

A flexible token-driven system that let the company spin up new brands in weeks instead of quarters. One source of truth across web, native, and CMS-driven landing pages.

// BRIEF

The company runs several consumer brands on a shared technical platform. Each brand had drifted into its own component library, its own colors, its own opinions. Launching a new brand took two quarters of design and engineering work — and the existing ones were slowly diverging from each other.

I joined as Design Lead to build a system that could host all brands at once: same components, swappable identity, no duplicated work.

Brands
4 live, 3 in flight
Surfaces
Web, iOS, Android, CMS

Status
● Shipping

// APPROACH

We pulled the brand layer out of the components and pushed it down into tokens — color, type, radius, motion. Components stopped caring which brand they were rendering. A new brand became a JSON file plus three reviews.

  • Audited 11 existing component libraries across brands and platforms.
  • Defined three token layers — primitive, semantic, brand — so designers could change wallpaper without touching plumbing.
  • Rebuilt 60+ core components in Figma + Storybook with parity across iOS, Android and Web.
  • Set up brand-switcher previews in every PR so reviewers see all four brands at once.
